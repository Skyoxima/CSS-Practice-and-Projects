⚝ By default all HTML elements have the position property as 'static'. It is the regular flow of positioning in HTML.

⚝ Relative position is similar to static but it allows for using the 'top', 'left', 'right' and 'bottom' properties.
These properties make the element dislocate from it's static position to whatever is specified.
Relative positioning causes the original space to still be taken up. This can cause overflowing of one element onto the next one.

⚝ Absolute position takes the element out of the normal document flow completely. No space is left in place of the actual element like it does in relative positioning.

Absolute position by default 'absolutely' positions the element it is applied to. By that it means that it will take the 'body' as the 'parent' of the element.
To have a absolutely positioned child element inside a parent element, the parent element has not to be static but anything else (relative, absolute, sticky, fixed).

⚝ Fixed position works and behaves like Absolute position only thing is that when the page is scrolled, the element still stays right where it was initially throughout the page.

⚝ Sticky positioning is essentially a combination of Relative and Fixed positioning. The element will be relatively positioned initially and then becomes fixed when the element is at the top of the parent while scrolling.
